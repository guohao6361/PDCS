# 修改文档

## [1.3.0] - 2026-07-13

### 新增功能

- **商家余额自动结算**：订单状态从 DELIVERED 变为 COMPLETED 时，自动将订单金额增加到商家余额
- **前端余额实时同步**：页面加载/刷新时自动从服务器获取最新用户信息（余额等），无需重新登录
- **支付前商品有效性验证**：用户支付订单前验证商品是否存在、库存是否充足，防止商品下架后仍能支付

### 修复问题

- **商家余额增加接口被拦截**：`JwtAuthFilter` 的 `isInternalPath` 方法缺少 `/users/\d+/add-balance` 路径，导致内部服务调用返回 401
- **用户评价功能不可用**：nginx 配置缺少 `/reviews` 代理规则，导致评价请求返回"网络异常"
- **默认地址唯一性失效**：前端添加/编辑/删除地址后未重新加载地址列表，导致多个地址同时显示为默认
- **部署后端口转发停止**：每次部署后 `kubectl port-forward` 进程可能停止，导致网页无法访问

### 变更文件

#### user-service
- `UserService.java` — 新增 `void addBalance(Integer userId, BigDecimal amount)` 接口方法
- `UserServiceImpl.java` — 新增 `addBalance()` 实现方法，增加用户余额
- `UserController.java` — 新增 `PUT /{userId}/add-balance` 内部 API 端点（`@InternalApi`）
- `config/JwtAuthFilter.java` — `isInternalPath()` 方法添加 `/users/\d+/add-balance` 路径到白名单

#### order-service
- `service/impl/OrderServiceImpl.java` — 两处修改：
  1. `payOrder()` 方法：添加商品有效性验证（检查商品是否存在、库存是否充足）
  2. `updateOrderStatus()` 方法：订单完成时调用 user-service 的 add-balance API 增加商家余额

#### frontend
- `src/context/AuthContext.jsx` — 添加初始化后自动调用 `refreshUser()` 获取最新用户信息
- `src/pages/Profile.jsx` — 地址管理功能重构：
  - 新增 `loadAddresses()` 函数从服务器重新加载地址列表
  - `handleSaveAddress()` 保存后重新加载地址列表
  - `handleSetDefault()` 设置默认后重新加载地址列表
  - `handleDeleteAddress()` 删除后重新加载地址列表
- `nginx.conf` — 新增 `/reviews` 代理规则，转发到 product-service

### 部署说明

1. **user-service**：需重新编译部署，包含 addBalance 接口和 JwtAuthFilter 修复
2. **order-service**：需重新编译部署，包含商品验证和商家余额增加逻辑
3. **frontend**：需重新构建部署，包含余额自动刷新、地址管理修复、nginx 代理配置

### 数据库变更

- 无数据库结构变更
- MongoDB 操作：将所有商品的 `merchantId` 设为 `null`（测试数据无归属化）

### 接口变更

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| PUT | `/users/{userId}/add-balance` | 增加用户余额（内部调用） | InternalApi |

### 业务流程变更

**订单完成资金结算流程：**
```
用户确认收货 → 订单状态 DELIVERED → COMPLETED
    ↓
order-service 调用 user-service PUT /users/{merchantId}/add-balance
    ↓
商家余额增加订单金额
```

**支付前商品验证流程：**
```
用户发起支付 → 验证商品是否存在 → 验证库存是否充足
    ↓ (任一验证失败)
返回错误信息，阻止支付
```

## [1.2.0] - 2026-07-12

### 新增功能

- **商家商品管理权限控制**：商家只能修改/删除自己发布的商品，管理员可管理所有商品
- **商品无归属化**：所有测试商品的 `merchantId` 设为 `null`，仅管理员可管理

### 修复问题

- **商家后台显示他人商品**：后端 API `/products/merchant/{merchantId}` 正确过滤，前端调用已修正
- **商品图片无法显示**：确认 MinIO 存储路径与 Nginx 代理配置一致

### 变更文件

#### product-service
- `ProductController.java` — 修改/删除商品时添加商家权限校验
- `ProductServiceImpl.java` — 商品删除逻辑优化

#### frontend
- `src/pages/MerchantDashboard.jsx` — 调用 `getMerchantProducts(user.id)` 获取商家自己的商品
- `src/api/product.js` — `getMerchantProducts(merchantId)` 调用 `/products/merchant/{merchantId}`

## [1.1.0] - 2026-07-10

### 新增功能

- **用户认证安全**：密码使用 BCrypt 加密存储，替代明文存储
- **密码复杂度校验**：注册时校验密码最小长度（默认6位，可配置）
- **API 接口文档**：新增 `Docs/API接口文档.md`，包含路由总表、各服务接口详情、请求/响应示例
- **order-service 完整实现**：订单创建、用户订单列表、订单详情接口
- **JWT 鉴权过滤器**：cart-service、product-service、order-service 均添加 JWT 鉴权支持
- **跨服务 JWT 转发**：order-service 通过 RestTemplate 拦截器自动转发当前请求的 JWT Token

### 修复问题

- **product-service 公开接口被拦截**：商品浏览接口 (`/products/*`) 不再需要 JWT 鉴权
- **JJWT 废弃 API**：所有服务的 JWT 解析从 `setSigningKey()`/`getBody()` 升级为 `verifyWith()`/`getPayload()`
- **Spring Boot 包扫描范围**：所有服务的 `@SpringBootApplication` 添加 `scanBasePackages`，确保公共模块 `com.ecommerce.common` 被正确扫描
- **cart-service Lombok 编译失败**：移除 Lombok 依赖，改为手写 getter/setter
- **配置外部化**：默认余额、密码最小长度等配置项从硬编码迁移到 `application.yml`
