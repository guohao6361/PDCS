## ⚠️ 已知问题和解决方案

### 问题1: SSH密码认证失败
**现象**: `Permission denied, please try again`  
**原因**: SSH配置被修改，PasswordAuthentication可能被设为no  
**解决**: 通过VNC登录，修改 `/etc/ssh/sshd_config`，确保：
```
PasswordAuthentication yes
PermitRootLogin yes
```
然后重启SSH: `systemctl restart sshd`

### 问题2: 镜像源超时
**现象**: `TLS handshake timeout` 或 `403 Forbidden`  
**原因**: 中国大陆网络环境限制  
**解决**: 使用 `docker save/load` 方式绕过网络拉取

### 问题3: minikube image load卡住
**现象**: 命令执行后一直等待，无输出  
**原因**: Minikube内部Docker与宿主机通信问题  
**解决**: 使用以下方法替代：
```bash
docker save <image>:<tag> -o /tmp/<image>.tar
docker cp /tmp/<image>.tar <minikube-container-id>:/var/lib/docker/tmp/<image>.tar
docker exec <minikube-container-id> docker load -i /var/lib/docker/tmp/<image>.tar
```

### 问题4: 所有外部Docker镜像源不可用
**现象**: `Client.Timeout exceeded` 或 `TLS handshake timeout`  
**原因**: 中国大陆网络环境限制，所有镜像源（daocloud、1ms.run、阿里云、腾讯云）均不可用  
**解决**: 
1. 在本地Windows下载镜像并保存为tar文件
2. 上传到服务器
3. 使用上述方法加载到Minikube
4. 或者暂时跳过中间件部署，先测试应用服务
