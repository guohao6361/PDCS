```bash

scp ecommerce-project/user-service/src/main/java/... root@8.163.25.118:/root/ecommerce-project/user-service/src/main/java/...

# 2. 在服务器上编译、构建、部署

# 3. 启动端口转发（后台运行）

```

️ **注意**：
- 镜像tag不变时 `kubectl set image` 不会触发Pod重建，需执行 `kubectl rollout restart deployment/<name>` 强制重启
- 前端源码修改后必须重新执行 `deploy-frontend.sh`（服务器端 `npm run build`）
- 端口转发进程可能因Pod重启而断开，需重新启动

## ⚠️ 已知问题和解决方案

### 问题1: MinIO Service名称与代码配置不一致
**现象**: 头像上传显示成功但图片无法显示  
**原因**: K8s Service名为 `minio-service`，但代码中 `MINIO_ENDPOINT=http://minio:9000`，DNS解析失败  
**解决**: 将Service名改为 `minio`，同步更新 `user-deployment.yaml` 中 `MINIO_ENDPOINT` 为 `http://minio:9000`

### 问题2: `/users/files/` 被JWT过滤器拦截
**现象**: 图片上传成功但浏览器无法加载（返回401）  
**原因**: JWT过滤器对所有 `/users/*` 路径要求token，但浏览器加载图片不带token  
**解决**: 在 `JwtAuthFilter` 中将 `/users/files/` 路径设为公开访问

### 问题3: 前端源码修改后未重新构建
**现象**: 部署后新功能不生效  
**原因**: 只上传了配置文件（nginx.conf），未上传源码（image.js），服务器 `npm run build` 用的是旧代码  
**解决**: 确保所有修改的源码文件都上传到服务器后再执行 `deploy-frontend.sh`

### 问题4: 镜像tag不变时Pod不更新
**现象**: `deploy-single.sh` 执行成功但代码未生效  
**原因**: `kubectl set image` 在tag不变时不触发Pod重建  
**解决**: 执行 `kubectl rollout restart deployment/<name>` 强制重启Pod

### 问题5: 端口转发进程断开
**现象**: 服务部署后无法访问  
**原因**: Pod重启导致 port-forward 进程终止  
**解决**: 重新启动端口转发：`nohup kubectl port-forward service/web-service 30080:80 --address 0.0.0.0 > /tmp/port-forward.log 2>&1 &`

### 问题6: MinIO emptyDir数据丢失
**现象**: Pod重启后之前上传的图片消失  
**原因**: MinIO使用 `emptyDir` 卷，Pod重建后数据清空  
**解决**: 重新上传头像即可。如需持久化，建议改用PVC
