## 课设说明

**适用对象**：计算机科学与技术 / 软件工程 / 大数据 专业

**建议时长**：1 周（或 16 学时）

**运行环境**：macOS / Linux / Windows (WSL2)

**课设目标：**

1.  **掌握微服务拆分思想**：理解电商场景下，为何要将用户、商品、购物车、订单拆分为独立服务。

2.  **掌握异构数据库选型**：

    - 使用 **Redis**​ 处理高并发、低延迟的**购物车**数据。

    - 使用 **MySQL**​ 处理需要事务一致性的**订单与支付**数据。

    - 使用 **MongoDB**​ 存储结构灵活的**商品详情与评论**数据。

3.  **掌握容器化和Kubernetes单机部署**：能够在 **Minikube (单机
    Kubernetes)**​ 环境下，使用 YAML 文件部署完整的微服务应用。

4.  **加分点完成情况：**

    - Redis、MongoDB的集群部署：对应页码xx-xx

    - Ingress-Nginx入口流量负载均衡：对应页码xx-xx

    - 评价管理：对应页码xx-xx
