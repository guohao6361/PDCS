## 1. 支付订单

**请求方式：** `POST`

**请求路径：** `/orders/{id}/pay`

**路径参数：**
| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 订单ID |

**请求头：**
| 参数名 | 类型 | 说明 |
|--------|------|------|
| Authorization | String | Bearer {JWT Token} |

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "userId": 1,
    "totalPrice": 299.00,
    "status": "PAID",
    "createdAt": "2026-07-11T10:00:00"
  }
}
```

**业务逻辑：**
- 将订单状态从 `CREATED` 更新为 `PAID`
- 扣减用户余额
- 仅 `CREATED` 状态的订单可支付

## 3. 商品分批加载（分页查询）

**需求描述：**  
前端商品列表页应分批请求商品数据，而非一次性全量加载，以提升页面加载速度和用户体验。

**后端接口（已实现）：** `GET /products?page={page}&size={size}`

**请求参数：**
| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| page | int | 0 | 页码（从0开始） |
| size | int | 10 | 每页数量 |

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "content": [
      { "id": 1, "name": "商品A", "price": 99.00, "category": "电子" }
    ],
    "totalElements": 50,
    "totalPages": 5,
    "number": 0,
    "size": 10,
    "first": true,
    "last": false
  }
}
```

**前端实现要求：**
- 首次加载请求第 0 页（如 8 条）
- 滚动到底部或点击「加载更多」时请求下一页
- 根据 `last` 字段判断是否还有更多数据
- 搜索过滤仅对已加载数据进行客户端过滤

**当前状态：** ✅ 前端已实现分批加载（首次加载 8 条，点击「加载更多」请求下一页）

## 4. 商品搜索（全量过滤）

**需求描述：**  
前端搜索功能需对**全部商品**进行过滤，而非仅对已加载的商品进行过滤，确保搜索结果完整。

**后端接口需求（未实现）：** `GET /products/search?keyword={keyword}&page={page}&size={size}`

**请求参数：**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| keyword | String | 是 | 搜索关键词（匹配商品名称或分类） |
| page | int | 否 | 页码，默认 0 |
| size | int | 否 | 每页数量，默认 10 |

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "content": [
      { "id": 1, "name": "商品A", "price": 99.00, "category": "电子" }
    ],
    "totalElements": 5,
    "totalPages": 1,
    "number": 0,
    "size": 10,
    "first": true,
    "last": true
  }
}
```

**当前问题：**  
前端搜索仅对已加载的商品数据进行客户端过滤，若商品未加载则无法被搜索到。

**解决方案：**  
后端提供搜索接口，支持按关键词查询全部商品并返回分页结果。

**当前状态：** ❌ 后端未实现搜索接口，前端仅支持客户端过滤
