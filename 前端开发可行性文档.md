# 前端开发可行性文档

## 1. 概述

本项目为并行计算课程设计「云原生微服务电商系统」的前端部分。现有系统已包含 4 个 Spring Boot 后端微服务（user/product/cart/order），分别连接 MySQL、MongoDB、Redis 等异构数据库。前端需要从原有的单 HTML 页面升级为 React SPA，提供完整的电商用户体验，包括用户注册登录、商品浏览、购物车管理、订单结算、在线支付和商品评价等功能。

## 2. 技术选型可行性

| 技术 | 版本 | 选型理由 |
|------|------|----------|
| React | 18 | 组件化开发、生态成熟、社区活跃，适合中大型 SPA |
| Vite | 5 | 基于 ES Module 的构建工具，开发启动快（<1s）、HMR 即时生效，远优于 Webpack |
| React Router | v6 | 官方推荐路由方案，支持嵌套路由、路由守卫，API 简洁 |
| Axios | 1.x | 支持请求/响应拦截器，可统一附加 JWT、处理错误格式，比 fetch 更易封装 |
| CSS Modules | 内置 | 样式隔离避免冲突，无需引入重型 UI 库（如 Ant Design），保持项目轻量 |

**不引入重型 UI 库的原因：** 本项目页面数量有限（6 个页面），使用 CSS + 设计规范即可满足 UI 需求，避免引入 Ant Design 等库导致打包体积过大。

## 3. 后端 API 对接可行性

### 3.1 统一响应格式

后端所有接口统一返回 `ApiResponse<T>` 结构，前端只需一套解析逻辑：

```json
{
  "code": 200,
  "message": "success",
  "data": { ... },
  "timestamp": 1752105600000
}
```

- `code = 200`：成功，`data` 为业务数据
- `code = 400`：业务校验失败，`message` 为错误原因
- `code = 401`：未认证，前端自动跳转登录页
- `code = 404`：资源不存在
- `code = 500`：服务器内部错误

后端已定义完整 REST API（4 个服务共 50+ 个接口）：

### 3.3 前端请求封装

Axios 拦截器统一处理 JWT 附加和响应解析：

```javascript
// 请求拦截器：自动附加 JWT
axios.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器：统一解析 ApiResponse
axios.interceptors.response.use(
  response => {
    const { code, message, data } = response.data;
    if (code === 200) return data;
    if (code === 401) {
      localStorage.removeItem('token');
      window.location.href = '/login';
      return Promise.reject(new Error(message));
    }
    return Promise.reject(new Error(message));
  },
  error => Promise.reject(new Error('网络异常，请检查连接'))
);
```

### 3.4 分页支持

商品列表接口支持分页，前端通过查询参数控制：

```
GET /products?page=0&size=10
```

响应 data 结构：`{ content: [...], page: 0, size: 10, totalElements: 25, totalPages: 3 }`

### 3.5 关键数据结构

**User**（MySQL）：`{ id, username, balance, phone, email, avatar, role(USER/MERCHANT/ADMIN) }`
- `password`、`payPassword` 带 `@JsonIgnore`，接口不返回

**Product**（MongoDB）：`{ id, name, price, stock, category, merchantId, description, imageUrl, attributes }`
- `attributes` 为动态字段 Map，不同商品可有不同属性键值对

**Address**（MySQL）：`{ id, receiverName, phone, province, city, district, detailAddress, isDefault }`

**Review**（MongoDB）：`{ id, productId, username, content, rating(1-5) }`

**CartResponse**（Redis）：`{ userId, items: [{ productId, quantity }] }`
- 购物车只存 productId + quantity，前端需根据 productId 关联商品信息

**OrderResponse**（MySQL）：`{ id, userId, merchantId, addressId, totalPrice, status(UNPAID/PAID/SHIPPED/IN_TRANSIT/DELIVERED/COMPLETED/CANCELLED), items: [{ productId, productName, price, quantity }], createdAt }`

## 5. 状态管理方案

| 状态 | 管理方式 | 持久化 |
|------|----------|--------|
| JWT Token | AuthContext（React Context） | localStorage |
| 用户信息（userId, username, balance, role） | AuthContext | localStorage |
| 购物车数据 | 页面组件 state（每次从后端拉取最新） | 不持久化（以服务端 Redis 为准） |

**不引入 Redux/Zustand 的原因：** 本项目状态简单，全局状态仅有登录态，其余数据均为服务端数据，每次页面加载时从 API 获取即可，无需复杂状态管理。

## 6. 本地开发环境搭建

### 6.1 前置条件

| 依赖 | 版本要求 | 用途 |
|------|----------|------|
| Node.js | >= 18 | 运行前端开发服务器 |
| MySQL | 8.x | user-service / order-service 数据库 |
| MongoDB | 6.x | product-service 数据库 |
| Redis | 7.x | cart-service 数据库 |

启动前端前，需先确保后端 4 个微服务均已运行：

```
user-service    → localhost:8081
product-service → localhost:8082
cart-service    → localhost:8083
order-service   → localhost:8084
```

### 6.2 初始化项目

```bash
cd ecommerce-project
npm create vite@latest frontend -- --template react
cd frontend
npm install
npm install axios react-router-dom
```

### 6.3 配置 Vite 代理（解决跨域）

开发阶段前端运行在 `localhost:5173`，后端在 `localhost:808x`，需配置代理：

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 5173,
    proxy: {
      '/users':    { target: 'http://localhost:8081', changeOrigin: true },
      '/products': { target: 'http://localhost:8082', changeOrigin: true },
      '/reviews':  { target: 'http://localhost:8082', changeOrigin: true },
      '/cart':     { target: 'http://localhost:8083', changeOrigin: true },
      '/orders':   { target: 'http://localhost:8084', changeOrigin: true },
    }
  }
})
```

### 6.4 启动前端

```bash
cd ecommerce-project/frontend
npm run dev
```

浏览器访问 `http://localhost:5173` 即可看到页面，API 请求通过 Vite proxy 自动转发到后端。

### 6.5 本地开发流程

```
1. 启动 MySQL / MongoDB / Redis
2. 依次启动 4 个后端微服务（mvn spring-boot:run 或 IDE 运行）
3. 启动前端（npm run dev）
4. 浏览器访问 http://localhost:5173
5. 修改代码后 Vite HMR 自动刷新，无需手动重启
```

## 7. 部署可行性

### 7.1 Docker 多阶段构建

```dockerfile
# 阶段一：Node 构建静态文件
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# 阶段二：Nginx 托管静态文件 + 反向代理
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
```

### 7.2 Nginx 配置

```nginx
server {
    listen 80;

    # 静态文件托管
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;  # SPA 404 兜底
    }

    # API 反向代理（按路径前缀分发到各微服务）
    location /users/  { proxy_pass http://user-service:8081; }
    location /products/ { proxy_pass http://product-service:8082; }
    location /reviews/ { proxy_pass http://product-service:8082; }
    location /cart/    { proxy_pass http://cart-service:8083; }
    location /orders/  { proxy_pass http://order-service:8084; }
}
```

### 7.3 部署兼容性

| 部署方式 | 配置 |
|----------|------|
| Minikube（K8s） | web-deployment.yaml（前端镜像）+ Service 暴露 80 端口 |
| Linux 服务器（Docker Compose） | docker-compose.yml 中 frontend 服务，依赖各微服务 |

两种方式均使用同一前端 Docker 镜像，无需修改代码。

## 8. 风险分析与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 跨域问题 | API 请求被浏览器拦截 | Nginx 反向代理统一同源，彻底避免 CORS；开发阶段 Vite 配置 proxy 代理 |
| React Router 刷新 404 | 用户刷新页面白屏 | Nginx 配置 `try_files $uri $uri/ /index.html` 兜底 |
| JWT 过期体验中断 | 用户操作中途被跳转登录 | Axios 拦截器统一捕获 401，弹出提示后跳转登录页 |
| 后端接口未就绪 | 前端开发阻塞 | 先用 Mock 数据开发，定义好 API 模块接口，后端就绪后仅需切换 baseURL |
| 移动端适配 | 页面在小屏设备显示异常 | CSS Flexbox 布局 + 媒体查询响应式断点 |
| 构建产物过大 | 首屏加载慢 | Vite 默认 Tree Shaking + Code Splitting；按需加载页面组件 |
