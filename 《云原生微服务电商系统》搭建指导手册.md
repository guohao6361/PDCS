# **一、引言**

本教程及课程设计成果从零开始，在 Windows (WSL2) + Linux
交叉环境下，完整落地并打通一整套电商微服务系统的基础设施与核心服务链。具体实现内容如下：

- 环境准备与基础设施搭建

- 统一反向代理网关与 Web 服务器部署

- 购物车微服务（接入 Redis 缓存）

- 用户微服务（接入 MySQL 数据库）

- 商品与评价微服务（接入 MongoDB 数据库）

- 采用docker+K8s部署微服务

本教程已搭建好整套系统的技术骨架，但订单管理微服务仍需同学们自由发挥，可考虑在此教程基础上开发。供参考的加分点如下：

- Redis、MongoDB的集群部署

- K8s的pod管理与数据持久化（StatefulSet替代Deployment）

- Ingress-Nginx入口流量负载均衡

- 评价管理

- 推荐功能

- 鉴权功能

# **二、常用的操作指南（供后续快速查阅，初次阅读请跳过）**

（1）环境初始化与集群管理

1、创建新的K8s集群

minikube start \--cpus=4 \--memory=4096 \--disk-size=20g
\--driver=docker \\ \--image-mirror-country=\'cn\' \\
\--image-repository=\'registry.cn-hangzhou.aliyuncs.com/google_containers\'
\\ \--base-image=\'gcr.io/k8s-minikube/kicbase:v0.0.50\'

2、如果电脑卡死，在window powershell运行此命令一键强制释放整个wsl内存：

wsl \--shutdown

3、删除K8s集群

minikube delete

4、查看本地Docker容器状态（验证Docker通道是否正常）

docker ps

（2）微服务编译打包、构建与镜像加载

每次修改java源码或配置文件后，都需要按顺序执行下面4个步骤，才能将最新代码送进K8s。

1.  切换到对应的微服务目录（xxx为微服务名称）

cd \~/ecommerce-project/xxx-service

2.  Maven编译

mvn clean package

3.  构建本地Docker镜像（强烈推荐使用递增的版本标签如 v1, v2,
    v3，彻底避开 K8s 镜像缓存）

docker build -t xxx-service:v2 .

4.  将本地构建的镜像推入并缓存到Minikube集群内部

minikube image load xxx-service:v2

（3）Kubernetes资源编排与动态修改（使用配置清单控制服务部署，或在不重新打包代码的情况下动态调整运行参数。）

1、应用/更新配置文件

kubectl apply -f \~/ecommerce-project/k8s/xxx-deployment.yaml

2、动态更新部署镜像（比如将购物车服务一键升级为v2镜像）

kubectl set image deployment/cart-service cart-service=cart-service:v2

3、重启服务

kubectl rollout restart deployment/xxx-service

（4）状态诊断、日志查看

1\. 查看所有 Pod 的运行状态（READY 必须为 1/1）

kubectl get pods

2、查看服务的端点绑定（非常重要：验证 Service 是否对准并成功绑定了底层的
Pod IP）

kubectl get ep xxx-service

3、诊断 Pod 异常（查看其生命周期事件，例如挂载失败、ImagePullBackOff
等具体原因）

kubectl describe pod \<pod-name-or-selector\>

5.  追踪 Java 容器控制台输出

kubectl logs -f -l app=user-service

（5）端口转发、数据库登陆与验收验证

1\.
启动网页网关反向代理端口转发（整个系统只需开启这一个转发，访问http://localhost:8080
即可）

kubectl port-forward service/web-service 8080:80

2、穿透登录进入 K8s 内部的 Redis 数据库，查看购物车

kubectl exec -it \$(kubectl get pods -l app=redis -o
jsonpath=\'{.items\[0\].metadata.name}\') \-- redis-cli

\# 交互界面命令：

\# \> KEYS \*

\# \> HGETALL cart:user:1001

\# \> exit

3、穿透登录进入 K8s 内部的 MySQL 数据库，查看注册用户

kubectl exec -it \$(kubectl get pods -l app=mysql -o
jsonpath=\'{.items\[0\].metadata.name}\') \-- mysql -uroot
-prootpassword

\# 交互界面命令：

\# \> use ecommerce;

\# \> select \* from users;

\# \> exit

4\. 在终端测试注册接口

curl -X POST http://localhost:8080/users/register \\ -H \"Content-Type:
application/json\" \\ -d
\'{\"username\":\"test888\",\"password\":\"123\"}\'

# **五、进阶开发指南**

1、总步骤：

**第一步（本地验证）：在 IDEA 里编写 Java 代码并直接在 IDEA
运行，连接本地 Docker
启动的轻量级数据库，用本地接口一键调试通。（保障没有逻辑
Bug）。亦可使用第三方Rest客户端进行测试。**

**第二步（构建镜像）：本地用 Maven 编译出 .jar 包，然后使用 docker build
封装成带版本号（例如 v1、v2）的 Docker 镜像。**

**第三步（推入集群）：运行 minikube image load
将本地刚刚构建好的镜像直接送入 K8s 集群的节点缓存中。**

**第四步（修改部署）：在对应的 K8s YAML（如 cart-deployment.yaml）中，把
image 里的镜像版本改到最新，运行 kubectl apply 或重启命令，让 K8s
瞬间无缝部署上线。**
